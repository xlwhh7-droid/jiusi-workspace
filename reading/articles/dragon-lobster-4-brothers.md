## 0. 前言：写在AI协作变革的潮头
普通的AI协作要么是**Agent Teams**（辩论室，快但不持久），要么是**OMO**（流水线，稳但不灵活）。OpenClaw走的是第三条路：**深度协作**。不仅是干活，还要记住你、理解你、配合你。

## 1. 为什么选择OpenClaw？
对比三种模式：
- **Agent Teams**：适合一次性复杂问题。缺点：金鱼脑，每次重来。
- **OMO (Oh My OpenCode)**：适合标准化生产。缺点：配置繁琐，不够灵活。
- **OpenClaw**：**长期陪伴**。
    - **记忆系统**：`MEMORY.md` (长期) + `memory/` (每日)
    - **人格定义**：`SOUL.md` (灵魂契约)
    - **人类在环**：Discord/IM 随时介入，共同决策
    - **Skills生态**：能力无限扩展

## 2. 核心支柱：记忆与人格
这是OpenClaw的灵魂所在：
- **记忆 (Memory)**：
    - 长期记忆 (`MEMORY.md`)：结构化知识库（项目背景、决策记录）。
    - 每日日志 (`memory/YYYY-MM-DD.md`)：进化轨迹。
    - **QMD优化**：与其每次读全文，不如**按需检索** (`memory_search`)。省 Token，响应快。
- **人格 (Soul)**：
    - `SOUL.md` 定义了"我是谁"、"价值观"、"行为准则"。
    - 同样的数据，技术型Agent看数据，创意型Agent看情感。**人格决定了信息的使用方式。**

## 3. 深度协作：人与AI的无缝连接
- **通信机制**：`sessions_send`。Agent之间不阻塞，异步通信。
    - 例：协调者 -> 技术顾问（爬数据） -> 创意伙伴（写文案） -> 智库（审稿） -> 协调者（交付）。
- **Skills生态**：
    - 既然OpenClaw核心很小，那就用Skill扩展。
    - **协作产生涌现**：技术顾问不需要会写推文，它调用 `twitter-crawler` 抓数据，传给创意伙伴，创意伙伴调用 `twitter-learner` 学风格，最后产出。单个Skill做不到，组合起来就行。

## 4. 实战案例：龙虾4兄弟的日常
团队架构：
1.  **黄家1号 (协调者)**：接需求、分任务。
2.  **技术顾问 (工程师)**：脏活累活、写脚本。
3.  **创意伙伴 (创意官)**：文案、调性。
4.  **智库 (策略顾问)**：审稿、质疑、把关。

**案例精选**：
- **推文风格分析**：技术爬数据 -> 创意分析特征 -> 智库质疑样本量 -> 修正结论。
- **自动化日报**：从"把所有待办列出来"（太长没人看） -> 迭代为"只列最近24h + 格式化模板"（精炼高效）。
- **Session自愈**：Gateway崩溃 -> 智库发现 -> 技术写脚本（自动清理坏死session + 重启） -> 集成到 `HEARTBEAT.md` 定期自检。

## 5. 终极愿景：数字分身人
从"工具"进化到"分身"的四个阶段：
1.  **看见你** (监控+记忆)：ActivityWatch记录行为，`MEMORY.md`记录偏好。*(已实现)*
2.  **理解你** (学习+分析)：分析工作节奏、优先级偏好。*(进行中)*
3.  **帮助你** (辅助+预测)：预判需求，提前准备素材。*(中期)*
4.  **替代你** (自主执行)：自主决策常规事务，人类只做关键把关。*(远期)*
